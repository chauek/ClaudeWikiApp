## Highlights

- Todos now have **Priority** and **Size** fields, visible as a colored badge and a chip.
- Tasks view adds a new **"By priority"** grouping (now the default), alongside the existing by-section grouping.
- Graph view got visual polish and a few behaviour tweaks.

## Features

- **Todo priority & size.** Every todo can be tagged with a priority (`critical` / `high` / `medium` / `low` / `someday`) and a size (`S` / `M` / `L` / `XL`). Both are editable inline via dropdowns in the Tasks view and shown as read-only indicators in the Node detail panel.
- **Priority-grouped Tasks view.** New view mode groups todos by priority, with per-priority coloured borders and badges. The view mode toggle sits next to the filter controls; "By priority" is the default view.
- **Localised labels.** Priority, size, "view by section", and "view by priority" strings are translated for both `en` and `pl`.

## Improvements

- **Graph view polish.** Refreshed node/edge styling and a handful of rendering fixes.
- **Scaffold CLAUDE.md.** Expanded the todo schema section so the knowledge base documents the new `priority` and `size` frontmatter fields.

## Under the hood

- New IPC handlers `fs:writeTodoPriority` and `fs:writeTodoSize`, exposed on `window.api` via the preload script.
- Shared `TodoPriority` / `TodoSize` types added to `types.ts` and threaded through `TodoInNode` and `TodoItem`.
- `pg-*` priority-group CSS utilities.

## Scaffold

- Scaffold version bumped to **2**. Existing knowledge bases will see the "scaffold outdated" notice in Settings — update to pick up the `priority` and `size` todo schema fields.
