## Highlights
- Drag tasks to reorder them within their priority or section group — the order is saved and persists across reloads.
- New `new` status for auto-created tasks, pinned at the top of the list so you can review and acknowledge before promoting them.
- Click a task description to expand the full text; a new arrow button jumps you to the source node, and clicking a task in a node detail now focuses and flashes it in the Todos view.

## Features
- **Drag-reorder tasks.** Each task gains an optional `sortOrder` (number) stored alongside `priority` / `size`. By Priority view → drag inside a priority group; By Section view → drag inside a section group, where dropping across priority bands also bumps the moved task's priority to match the row below the drop position. Order is written through to both `_meta/todos.json` and the source node's `.md` frontmatter, so it survives Claude-driven rebuilds.
- **`new` status** for auto-added tasks, rendered in a dedicated group above all priority buckets (including critical) so you can review before promoting to `pending`.
- **Click-to-expand task descriptions** (clamped to one line by default), a right-arrow nav button to open the linked node, and click-from-NodeDetail to focus + flash the matching task in the Todos view.
